Transiciones e iteraciones visuales

View Transitions es la herramienta visual de Astro para crear animaciones fluidas y mantener elementos vivos entre páginas sin recargar el navegador, dando la sensación de que el usuario está navegando en una aplicación móvil o una Single Page Application (SPA) súper rápida, aunque por detrás sigan siendo páginas HTML independientes.

Con las View Transitions activadas, Astro intercepta el clic a un enlace, descarga la nueva página en segundo plano, compara ambas páginas y anima el cambio (por ejemplo, haciendo un desvanecimiento suave o fade).

Hero Transitions

Las Hero Transitions permiten la animación de elementos compartidos entre dos páginas. Ejemplo:Si se tiene una imagen de un gatito en una tarjeta pequeña en el index.html y se entra a ver eldetalle de la tarjeta en otra página, se puedes hacer que esa misma imagen vuele y se agrande suavemente hasta su nueva posición en la segunda página. Solo se necesita poner el mismo nombre: transition:name="mi-gatito".

Persistencia de Estado

La Persistencia del estado (transition:persist), permite mantener en memoria un archivo multimedia que se esté reproduciendo al cambiar de página. El elemento con persistencia de estado, no se recarga ni se corta. Ejemplo: la música sigue sonando mientras la página cambia.

Control de Dirección (fallback y animaciones direccionales)

Astro hace por defecto un desvanecimiento (fade), pero se pueden crear animaciones que entiendan hacia dónde va el usuario.

Si se tiene una lista de ejercicios y el usuario hace clic en Siguiente, se puede hacer que la nueva página entre deslizándose desde la derecha. Si pulsa el botón Atrás del navegador, Astro detecta la dirección y hace que la página anterior vuelva deslizándose desde la izquierda.

Esto se logra con la directiva transition:animate="slide". El navegador entiende el historial de navegación y adapta el movimiento automáticamente.

Animaciones Completamente Personalizadas (Custom Animations)

Además de los movimientos por defecto (fade, slide o none), se pueden definir animaciones personalizadas usando CSS nativo o propiedades de JavaScript y empaquetarlas en un objeto para que Astro las ejecute.

Se pueden definir cosas como:


const miAnimacionPersonalizada = {
  old: { name: 'unblur', duration: '0.3s', easing: 'ease-out' },
  new: { name: 'blur', duration: '0.3s', easing: 'ease-in' }
};

Esto permite, por ejemplo, que la página vieja se desenfoque poco a poco mientras la nueva aparece nítida desde el fondo.

Eventos del Ciclo de Vida

Cuando se usa View Transitions, la página nunca se recarga de verdad, lo que significa que el clásico evento de JavaScript document.addEventListener('DOMContentLoaded', ...) solo se ejecuta la primera vez que se entra a la web.

Si se cambia de página, los scripts (como por ejemplo los contadores) dejarían de funcionar. Para solucionar esto, la API de View Transitions de Astro ofrece un ciclo de vida con eventos especiales:

  • astro:before-preparation: Se ejecuta justo cuando se hace clic en un enlace, antes de que Astro descargue la siguiente página (ideal para poner una pantalla de carga o un spinner).
  • astro:after-swap: Se ejecuta en el milisegundo en que la página nueva reemplaza a la vieja.
  • astro:page-load: Reemplaza por completo a DOMContentLoaded. Se ejecuta cada vez que una página termina de cargar, ya sea la primera vez o tras una transición.

Navegación Programática (Hacer transiciones con código)

Astro permite disparar una transición de página fluida desde un código JavaScript usando el método Maps().

Esto permite que al llenar un formulario y, tras validar que los datos en la base de datos de SQL son correctos, se quiera redirigir al usuario a una página de éxito de forma fluida:


 import { navigate } from 'astro:transitions/client';

 // Redirigir de forma animada mediante código
 navigate('/ejercicios/exito');
