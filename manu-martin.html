Astro

Ventajas de Astro

Astro cuenta con las siguientes ventajas sobre otros Frameworks de JS:

  • Sencillez de su código
  • La rapidez
  • Funciona tanto solo, como integrar otros frameworks de JS
  • Variedad de Plantillas

Creando un proyecto con Astro

Necesitamos instalado en la máquina node.js

Para crear un proyecto nuevo en astro, entramos en la carpeta superior que contendrá la carpeta del proyecto, y ejecutamos el siguiente comando con la terminal (cmder): pnpm create astro@lastest

Al lanzar el comando y una vez instalado lo necesario, Astro nos hará las siguientes preguntas:

  1. ¿Dónde quieres crear tu aplicación? (Where should we create your new project)? Ejemplo: aprendiendoastro

  2. Luego, te preguntará cómo quieres comenzar el proyecto (How would you like to start your new project?) con las siguientes opciones para que elijas:

    • > Include sample files (incluye archivos de ejemplo)
    • - Use blog template (usa una plantilla de blog)
    • - Empty (proyecto vacío). Elije este tercer punto

  3. A continuación preguntará si se desea instalar las dependencias (Install dependencies?). Selecciona Yes (Sí)

  4. Luego pregunta si deseas programar en TypeScript [JS tipado] (Do you plan to write TypeScript?) En este caso, selecciona No.

  5. A la pregunta de si deseas inicializar un repositorio de github (Initialize a new git repository?) Responde No.

Una vez terminado el proceso, podremos abrir el proyecto con Visual Studio Code en la carpeta en la cual creamos el mismo en el Punto 1: (aprendiendoastro)

Al abrir la carpeta verás la siguiente estructura en VSCODE

  • > .vscode
  • > node modules: Donde se instalan los módulos del proyecto
  • > public: Aquí guardaremos los archivos públicos de la aplicación (en este caso los assetsvan acá)
  • > src: En src vamos a tener el directorio de páginas (pages) del proyecto. Su extensión es .astro
  • .gitignore
  • astroconfig.mjs: Archivo de configuración de Astro
  • package-lock.json
  • package.json: Manejar las dependencias del proyecto
  • README.md:
  • tsconfig.json: fichero de configuración de TypeScript (Astro internamente lo usa)

Instalaremos la extensión Astro de Visual Studio Code, para colorear el código de Astro.

Inicializar el proyecto de desarrollo

Ejecutamos pnpm run dev. Esto nos permitirá abrir un servidor en el navegador.

Componentes

Los componentes son como piezas de un puzzle que nos ayudan a reutilizar código de nuestra interfaz. En vez de tener todo el código en un archivo, se crean componentes que ayudan tanto a organizar el código como a hacerlo reutilizable.

Para guardar los componentes de la aplicación, se crea un directorio dentro de la carpeta src al cual llamaremos components. Los nombres de los archivos de componentes (archivos en src/components), así como los de los Layouts que se verán más adelante, se nombran con la primera letra de cada palabra en mayúscula (UpperCamelCase).

Vamos a crear un post para un blog. Para ello creamos un archivo al cual llamaremos Post.astro.

Los archivos .astro tienen una estructura un poco diferente a los de .html.

Los archivos .astro tienen una sección llamada frontmatter (la sección entre ---) en la parte superior de los mismos. El frontmatter se utiliza principalmente para definir metadatos y configuraciones específicas de cada archivo, especialmente importaciones, contenido Markdown, MDX o páginas .astro

Agregando Tailwind al proyecto con pnpm

Para agregar Tailwind al proyecto se ejecuta desde la terminal: pnpm astro add tailwind.

El asistente pedirá confirmación tanto para instalar tailwindcss como su archivo de configuración. Responde Y y presiona Enter. Esto instalará @astrojs/tailwind y tailwindcss, y actualizará el archivo astro.config.mjs.
