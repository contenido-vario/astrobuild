Plantillas (Layouts)

Los layouts nos ayudan a crear contenido que se va a repetir en todas las páginas o en las páginas que nosotros queramos sin tener que volver a hacerlo nosotros a mano, por ejemplo headers, footers, barras de navegación. En realidad un Layout es un tipo de componente estático.

Para trabajar con layouts, creamos una nueva carpeta en la carpeta src y la llamamos layouts. Crearemos un layout que será la estructura por defecto en todas las páginas del proyecto y lo llamaremos, DefaultLayout.astro.

Pondremos allí, todo el contenido que queremos que sea reutilizable en todas las páginas, excepto lo que es específico de ellas.

Le podemos agregar lo que deseemos. En este ejemplo agregaremos un <header> y un <main>. Dentro del <header> colocaremos una barra de navegación <nav> (Puedes copiar y personalizar luego, una barra de navegación de la página de Flowbite: https://flowbite.com/docs/components/navbar/).

Definir el layout que usará una página

Para definir el Layout o plantilla que queremos utilizar en una página, lo importo en el Frontmatter. En el área de html se coloca el código específico de cada página, dentro de la llamada al Layout. Para hacer la llamada a un Layout, se hace referencia a su nombre, usando la etiqueta de apertura y de cierre. Ejemplo:


---
import Post from "../components/Post.astro";
const response = await fetch("http://localhost:3000/posts");
const posts = await response.json();
import DefaultLayout from "../layouts/DefaultLayout.astro";
---
<DefaultLayout>
  <h1 class="text-center text-3xl font-bold mb-6 mt-3">Mis posts
  <div class="grid max-w-7xl mx-auto w-full justify-center justify-items-center items-center gap-4 px-2 sm:grid:cols-2 lg:grid-cols-3">
    {posts.map((post) => <Post post={post} />)}
  </div>
</DefaultLayout>

Como al layout, no le hemos dicho dónde tiene que mostrar el contenido de la página (en nuestro caso los posts), hay que indicarlo en el mismo. En Astro, al espacio que dejamos para el contenido que difiere en el Layout se conoce como un slot (en otros frameworks se llaman children).

Un paso más allá: named slots y valores por defecto en un slot

Los named slots se utilizan si queremos que varias partes dentro de un Layout varíen (no una como en el caso anterior). Para definir un named slot dentro de un Layout se usa la sintaxis:

<slot name="nombre-del-slot"/>

En este caso, debemos darle nombre a los slots que queremos pasar al Layout, dentro de cada página que utilice ese Layout. Por ejemplo: si quiero que cada página tenga un título diferente en el navegador, puedo crear un slot llamado titulo de la siguiente manera:


---
import Post from "../components/Post.astro";
const response = await fetch("http://localhost:3000/posts");
const posts = await response.json();
import DefaultLayout from "../layouts/DefaultLayout.astro";
---
<DefaultLayout>
  <title slot="titulo">AstroBlog</title>

  <h1 class="text-center text-3xl font-bold mb-6 mt-3">Mis posts
  <div class="grid max-w-7xl mx-auto w-full justify-center justify-items-center items-center gap-4 px-2 sm:grid:cols-2 lg:grid-cols-3">
    {posts.map((post) => )}
  </div>
</DefaultLayout>

Lo que esté dentro de las etiquetas <DefaultLayout></DefaultLayout> se muestra en el slot por defecto (aquel que no tiene nombre). Pero el título se mostrará en el <slot> título.

En DefaultLayout.astro dentro del <header> se especificaría que Astro renderiza la página con el contenido que tenga el slot cuyo nombre sea título.


<!DOCTYPE html>
<html lang="es-Es">
<head>
  <meta charset="UTF-8">
  <link rel="shortcut icon" type="image/svg+xml" href="favicon.ico" >
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="generator" content={Astro.generator}>
  <slot name="titulo">Astro</slot>
</head>
...

Astro, mostraría el título que le hayan enviado dentro del slot cuyo nombre sea título. Entre las etiquetas <slot name="titulo"> y <slot> se puede colocar un valor por defecto en caso de que no se envíe ningún valor.
